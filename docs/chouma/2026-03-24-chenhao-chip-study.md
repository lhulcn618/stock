## 这本书对筹码分布的核心定义

1. “筹码分布”本质上是“流通股票持仓成本分布”，反映的是当前仍然持有流通盘的投资者，在不同价位上的持仓数量。
2. 它不是简单累计全部历史成交，而是要估计“哪些旧筹码已经卖出、哪些还留在场内”。
3. 书里明确提出，更精确的算法应考虑两个因素：
   - 浮动盈亏
   - 持股时间
4. 书里同时也承认，出于计算量考虑，可以先用“相等抛出概率”替代真实抛出概率，误差在实战上通常可接受。
5. 作者明确反对“只按时间衰减历史成交”的粗糙算法，因为不同股票活跃度差异很大，单纯时间挥发不能反映真实换手。

## 书里最重要的分析框架

### 1. 密集与发散

- 筹码在狭窄价格空间集中，叫“密集”。
- 筹码在较宽价格空间展开，叫“发散”。
- 书里的判断很明确：
  - 密集往往对应新的多空博弈前奏。
  - 发散往往对应趋势正在展开。

### 2. 低位密集

- 低位密集不是“绝对低价”，而是相对于之前明显下跌后的低位区。
- 书里给了一个很强的经验门槛：从高位向低位转移，最好至少有 `6` 个跌停板左右的空间，才更接近“真正的低位密集”。
- 低位密集通常被解释为：
  - 高位套牢盘认赔离场
  - 新资金在低位吸筹
- 但作者也强调，不能机械地把“低位密集 = 立刻买入”，因为主力低位吸筹可能持续很久。

### 3. 高位密集

- 高位密集通常代表低位获利盘已经在高位大规模转移。
- 在作者体系里，它是明显的风险形态，经常对应主力出货或至少要高度警惕。
- 但也不是绝对教条，仍然要结合市场背景判断。

### 4. 获利盘/套牢盘

- 书里非常重视获利盘比例和套牢盘比例。
- 获利盘高说明当前价格之下的筹码较多，抛压类型和阶段判断会不同。
- 很多后续信号，例如“90 比 3”“无量上穿密集区”，都是围绕获利盘比例和换手率展开。

### 5. 主力成本不是一个孤立数字

- 书里虽然经常用“峰顶价”估计主力成本，但实际表达更接近“主力成本区”。
- 尤其在双峰、低位锁定、长期横盘这些场景下，作者更重视：
  - 低位峰是否是主力成本区
  - 当前价是否仍在主力成本区附近
  - 是否具备高控盘证据

## 与我们当前实现的对照

## 已经一致的部分

1. 当前实现不是简单时间衰减，而是基于每日换手做筹码迁移，这一点符合书里的核心原则。
2. 我们使用了“当前存量筹码”思路：旧筹码按换手衰减，新成交筹码进入新的价格桶，这与书里的仿真思路一致。
3. 我们用了复权均价近似每日成交筹码成本，这比直接拿收盘价更合理。
4. 我们已经输出了书里很关键的几个结果：
   - 主力成本
   - 平均成本
   - 获利盘比例
   - 70%/90% 筹码集中区
   - 筹码峰列表

## 当前实现与书中方法的主要差异

1. 我们现在使用的是“等抛出概率”近似。
   - 实现方式：所有历史筹码都按当日换手率同比例衰减。
   - 书中观点：更精确的算法应让抛出概率同时依赖“浮动盈亏 + 持股时间”。
   - 结论：我们现在是合理的一阶近似，但还不是书里追求的更精细版本。

2. 我们把“主力成本”直接定义成最大筹码峰价格。
   - 这在很多单峰结构里成立。
   - 但书里更强调“主力成本区”，不是只看单个峰顶。
   - 在双峰、宽峰、多次换手堆积时，单点主力成本会偏粗。

3. 我们当前没有把“低位密集 / 高位密集 / 发散”单独结构化输出。
   - 书里这三个形态是核心语言。
   - 目前软件只给了数值，没有给足形态解释层。

4. 我们还没有把书里的“控盘证据链”接完整。
   - 例如：
     - 无量上穿密集区
     - 90 比 3
     - 成本均线发散
     - 主力成本区附近横盘
   - 目前软件里这些信息是分散的，还没合成一套“控盘判断”。

## 对当前软件最有价值的结论

### 结论 1

当前版本已经可以作为“筹码位置识别工具”使用，但还不应把“mainCost”解释成严格意义上的真实主力建仓价。

更稳妥的表述应该是：

- `mainCost`：当前最大筹码峰价格
- `averageCost`：市场平均持仓成本
- `concentration70/90`：主要筹码分布区间

### 结论 2

如果要更贴近这本书，下一步最应该补的是“形态判断”和“控盘证据”，不只是继续堆更多静态数值。

优先级建议：

1. 输出 `低位密集 / 高位密集 / 发散 / 双峰` 标签
2. 输出 `主力成本区`，不要只给单点 `mainCost`
3. 输出 `获利盘 + 换手率 + 密集区穿越` 的联合判断
4. 再考虑升级为“盈亏 + 持股时间”的非均匀抛出概率模型

### 结论 3

书里的核心用途不是“看一个数”，而是“判断当前阶段”：

- 主力是否可能在吸筹
- 主力是否可能在出货
- 当前价是否贴近主力成本区
- 当前是低位锁定、向上发散，还是高位风险区

## 建议的软件升级方向

### A. 新增“筹码形态”字段

建议在个股信息里新增：

- `chipShape`
- `chipRiskLabel`
- `chipControlEvidence`

其中 `chipShape` 可先支持：

- `低位密集`
- `高位密集`
- `向上发散`
- `向下发散`
- `双峰结构`
- `宽幅发散`

### B. 新增“主力成本区”

不要只显示一个 `mainCost`，建议增加：

- `mainCostZoneLow`
- `mainCostZoneHigh`
- `mainCostZoneSummary`

实现上可以先围绕最大峰左右累计到某个比例，得到一个峰区，而不是只取峰顶单点。

### C. 新增“控盘证据”

建议把下面几项做成结构化字段：

- 当前是否接近 `90 比 3`
- 是否出现“无量上穿密集区”
- 是否处于“低位锁定”
- 成本均线是否发散

这样个股弹窗里就能直接告诉你：

- 这只股票只是筹码好看
- 还是已经有较强的主力控盘证据

## 对当前实现的判断

当前实现方向是对的，且已经超过很多市面上“只做时间衰减”的简化版本。

但如果要严格对齐陈浩这套体系，当前仍然属于：

- `可用的一阶版本`
- `适合做持仓成本观察`
- `还不够做完整的主力行为判读`

## 后续建议

如果继续迭代，建议按这个顺序：

1. 先加“筹码形态标签”
2. 再加“主力成本区”
3. 再加“控盘证据面板”
4. 最后再做“非均匀抛出概率模型”

这样改动最稳，也最贴近这本书真正有用的部分。
